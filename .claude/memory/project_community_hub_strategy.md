The community-nurture funnel for AI Bible Gospels uses **Telegram channel** as the primary hub, **email list on aibiblegospels.com** as the durable foundation layer, and **Facebook Group** as the #2 social-discovery layer. Discord and TikTok community tools are explicitly out.

**Why:**
- Audience is 30+ Hebrew Israelite / prophecy / international diaspora — Telegram is already where IUIC-adjacent and prophecy-teacher communities live.
- Discord is wrong demo (Gen Z gaming) and high moderation cost for a solo operator.
- TikTok community tools defeat the purpose — the funnel goal is moving viewers OFF TikTok onto owned channels Tommy can't be deplatformed from.
- Solo operator can't actively nurture more than three places.

**Funnel architecture:**
```
TikTok viral hook
  ↓ pinned comment + bio link → aibiblegospels.com/#welcome
  ↓ email signup (prayer wall / scripture-drop hook)
  ↓ Telegram invite in welcome email
  ↓ Telegram: daily scripture + new YT alerts → drives YT views
  ↓ YouTube long-form (retention + watch hours for YPP reapply)
```

**How to apply:**
- All "where should we build community?" suggestions default to this stack. Don't propose Discord/Slack/Reddit/TikTok-community without a strong reason to override.
- Browser-scraped TikTok analytics feed the curation loop: which hooks earned funnel entry → topic of next clone batch.
- aibiblegospels.com repo is READ-ONLY (per feedback_repo_scope.md) — coordinate hub-link additions via the other Claude instance.
- Email list lives on the .com site as the foundation. It's the only platform that survives if any social account gets banned.
